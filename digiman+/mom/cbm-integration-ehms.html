Session 1 — 25 Mei 2026

1. Integration Points

  1. Integration via Topic: Digiman+ publish data ke topic (Btech), EHMS consume topic.
  2. EHMS consume semua topic meskipun data master belum available di EHMS.
  3. Jika ada equipment pindah site, EHMS perlu assess cara menampilkan data history CBM di dashboard.

2. Expected Flow — MO Service

  • MO Service sync to Digiman+ : status Release
  • Planner buat Planning di Digiplan, mapping MO Service terhadap planning tersebut
  • Eksekusi → proses input CBM
  • Execution finish
  • Publish data CBM ke topic (MO Service belum tentu sudah TECO)
  • EHMS consume topic → taruh ke table staging
  • EHMS mengolah data dari staging (assess lebih lanjut)

3. Notes & Constraints

  1. Di Digiman+ backlog execution baru activity type: BEX, belum ada EHM dan ACT. Future enhancement.
  2. TECO Date tidak dikirim dari Digiman+.
  3. SMU di Digiman+ belum ada validasi range perubahan setiap service-nya.
  4. Format date: ISO string.
  5. Mapping CBM Threshold dari transaksi Digiman+ akan ada di dalam Topic bersamaan value dan rating untuk transaksi tersebut.
  6. Data master CBM Threshold EHMS sync dari Digiman+ (perlu assess lebih lanjut).

    Sumber input data CBM:
    • Service Sheet (PSCS) : Digiman+
    • Uploader
    • EHM via uploader
    • ACT via uploader
    • Revise data CBM
    • Pengecekan under carriage
  7. Value dan rating data transaksi dari Digiman+ tidak perlu dikompare lagi di master data CBM Threshold EHMS. Jika memungkinkan, tampilkan reference master data threshold yang digunakan saat transaksi terjadi (perlu assess lebih lanjut).
Session 2 — 03 Juni 2026

4. Component & Equipment Mapping

  1. Mapping Equipment to Component di EHMS diambil dari SAP. Di Digiman+ dimaintain sendiri. Perlu disepakati dengan BPO flow setup mapping component jika suatu saat ada model baru.
  2. Digiman+ dapat mempertimbangkan membuat API untuk integrasi data component mapping (future).
  3. Monitoring CBM sampai level Component — tidak perlu sampai level sub component. Namun Digiman+ tetap mengirim data inputan user (CBM) sampai level sub component jika ada, beserta task key.

5. Data & Integration Scope

🔮 Future Journey
  1. Data master integration — future journey.
  2. Bank task integration — future journey.
  3. Follow up action integration / intervention — future journey.

6. Mapping & Validation Rules

  1. Area diisi dengan Sub Component dari Digiman+ jika ada.
  2. Mapping component to area jangan divalidasi saat integrasi — EHMS akan memproses dari table staging.
  3. CBM Parameter: Digiman+ copy dari EHMS untuk BUMA ID.

7. Task Key & Description

  1. Task Key EHMS masuk ke field external_code di Digiman+.
  2. Digiman+ kirim data transaksi ke EHMS — task key diambil dari external_code jika ada.
  3. Task Description untuk initial Digiman+ disamakan dengan EHMS.

8. CBM Task Types

3 tipe task CBM di EHMS saat ini:

Tipe Format / Nilai Keterangan
Manual Rating A, B, C, X Rating kondisi komponen
Measurement Decimal (2 digit) Nilai pengukuran numerik
Physical Condition Good, Damage, Missing Kondisi fisik komponen

9. Data Fields yang Dikirim Digiman+ ke EHMS

  1. UOM tetap dikirim dari Digiman+.
  2. IsSampleTaken: 1 jika data diisi oleh user, 0 selain itu.
  3. Rating dikirim beserta threshold yang digunakan saat transaksi.
